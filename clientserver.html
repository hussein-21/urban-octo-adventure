Client Server computing

by Hussein Alsaidi

What is the client/server model?

The client/server computing model, or architecture, is a computer/software network model in which some computers or software act as "clients" that make "requests" for a data or a service and other computers or software act as "servers" that listen for and respond to these requests. For example, a web client (e.g. a computer running the Chrome browser) makes requests for web pages from a web server (another computer running a web server software such as Apache).

Several major advantages of the client/server model

Several major advantages of the client/server model are:

  1. Centralized security and administration
  2. scalability (easy to add more servers and increase capabilities/power)
  3. Centralized client/user backup and recover
  4. User collaboration and shared resources, e.g. file server, printers, faxes, databases, etc.

Examples of TCP/IP (suite) protocals

  • FTP : protocol used to send/transfer/receive files from one computer to another (client to server and vice versa)
  • HTTP : protocol used to send/retrieve web pages over the Web using a Web browser
  • HTTPS : protocol used to send/retrieve web pages securely/encrypted over the Web using a Web browser
  • DHCP : protocol used to dynamically assign IP addresses to devices on the Internet
  • SMTP : used to send email messages

Examples of network utilities and services

  • ping : utility used to test whether a host on a network is reachable/available.
  • tracert : utility that displays the path packets are taking to get from your current location/device on a network to another location on a network. The number of hops and how long each hop takes is displayed allowing one to see if and where there are any delays between the two locations.
  • pathping : utility that combines the features of ping and tracert.
  • whois : utility that allows you to look-up who owns an Internet resource, such as a domain name or IP address block.
  • ipconfig : utility that allows you to look-up and configure a devices IP address, DHCP, and DNS information.
